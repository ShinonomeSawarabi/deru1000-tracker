# でる1000 トラッカー

TOEIC「でる順パート別問題集」（でる1000）の学習記録を管理するWebアプリです。

## 機能

- **15セクション対応** — 問題3〜659を15セクションに分けて管理
- **クール管理** — 複数回のくり返し学習（クール）をまとめて記録・比較
- **セクションごとの記録** — 実施日・誤答数・所要時間・最終残誤答を入力
- **ヘッダー統計** — 現在のクールの完了数・平均誤答・平均残誤答をリアルタイム表示
- **分析マトリクス** — クールをまたいだセクション別の誤答推移と改善トレンドを一覧表示
- **ローカル保存** — データはブラウザの localStorage に保存。インストール不要・サーバー不要
- **レスポンシブ対応** — PC・スマートフォンどちらでも使用可能

## 使い方

### 基本的な流れ

1. アプリを開くと **COOL 1**（1回目）が作成されています
2. サイドナビまたは画面上の `Sections` タブで学習したセクションを選択します
3. セクション行をクリックすると入力フォームが展開されます
4. **実施日・誤答数・所要時間・最終残誤答**を入力して `SAVE` を押します
5. 全15セクションを終えたら、設定ページ（⚙️）から **新しいクール** を追加して2周目を開始します

### 分析ページ

- ボトムナビの `Analysis`（📊）を開くと、クール別のセクションマトリクスが表示されます
- 上段が誤答数、下段が最終残誤答です
- 2クール以上になると **TREND** 列が表示され、初回と最終回の誤答数の増減がわかります

### データについて

データはブラウザの localStorage にのみ保存されます。**ブラウザのデータをクリアすると記録が消えます**のでご注意ください。

## GitHub Pages での公開

自分のリポジトリに fork してアクセスするには、以下の手順で GitHub Pages を設定します。

1. リポジトリの **Settings** → **Pages** を開く
2. **Source** を `Deploy from a branch` に設定
3. **Branch** を `main` / フォルダを `/ (root)` に設定
4. **Save** をクリック

公開後、以下のURLでアクセスできます。

```
https://<ユーザー名>.github.io/deru1000-tracker/
```
